This is how your token will be identified in wallets and exchanges. Example: Shardeum

The abbreviated ticker symbol representing your token, typically 3-5 letters. Example: SHD. Use uppercase letters by convention (e.g., SHD instead of shd) for better readability. Check if your symbol is already used on popular tokens to avoid confusion.

The number of tokens that are created (minted) right when the token contract is deployed. This is how many tokens exist initially. Example: If you set the initial supply to 1,000,000 tokens, then at deployment, exactly 1 million tokens will be minted and assigned to your address. Note: If your token is mintable, the initial supply is just the starting amount, you can increase total tokens later by minting. Important: Usually specified in the smallest divisible units (called 'token units' or 'base units'). For example, if decimals are 18, to mint 1 million tokens you enter 1000000 * 10^18 (1,000,000 followed by 18 zeros). Example: 1000000000000000000000000 (which equals 1,000,000 tokens with 18 decimals)

The maximum number of tokens that can ever exist. Even if minting is enabled, tokens cannot be minted beyond this cap.

How many decimal places your token supports. This determines how divisible the token is. The standard is 18 decimals, like Ether, meaning you can divide tokens down to 0.000000000000000001 units.

Indicates whether new tokens can be created (minted) after the initial deployment. Yes: You can create more tokens later. No: Token supply is fixed at initial supply, nobody can mint further tokens. (The Initial Distribution goes to your address.)
